What Is OpenClaw
03

OpenClaw 是什么

  • 一个运行在你自己设备上的个人 AI 助手框架。
  • 支持把 AI 接入你已经在用的消息渠道与工具系统。
  • 支持长期记忆、工作区、技能扩展和子智能体协作。
  • 适合做本地优先、可控制、可定制的教育助手。

Local-First
06

为什么“本地优先”对教育很重要

  • 教师更容易掌控数据去向与模型使用边界。
  • 学校资料、学生信息、教学文本不必天然暴露给第三方 SaaS。
  • 本地部署更利于在校内场景下建立可解释的技术链路。
  • 更适合后续做班级、教研组、校内平台的定制集成。

对学校而言，可控性往往比“最炫的功能”更重要。

这也是本地部署型智能体框架值得关注的原因。

教育视角数据、治理与可控性优先

Installation
08

安装准备：工作坊里最该讲清的技术前提

  • 官方 README 推荐运行时为 Node 24，或 Node 22.16+。
  • 推荐使用 `openclaw onboard --install-daemon` 进行引导式安装。
  • Onboard 会安装 Gateway 守护进程，使系统常驻运行。
  • 首次部署建议在个人电脑或可控测试机上完成，不要直接上正式教学环境。
# 推荐安装方式 npm install -g openclaw@latest # 或 pnpm add -g openclaw@latest openclaw onboard --install-daemon # 手动查看网关运行 openclaw gateway --port 18789 --verbose
部署准备先把运行时与守护进程概念讲明白

Always-On
10

为什么它能做到“7×24 小时在线”

  • 不是浏览器页面开着就算在线，而是有后台 Gateway 守护进程持续运行。
  • Onboard 安装的 daemon 会在 macOS / Linux 等环境中常驻。
  • 只要机器不断电、网络可用、渠道正常，智能体就可以持续接收与处理消息。
教育价值

教师下班后，助手仍可继续整理群消息、归档反馈、记录任务和等待指令。

这就是“从对话走向行动”的基础设施条件。
持续运行守护进程是关键，不是聊天窗口

Workspace

Security
13

安全与治理：工作坊一定要强调

  • OpenClaw 会接触真实消息渠道，因此所有外部输入都应视为不可信。
  • 官方 README 提示对 Telegram、WhatsApp、Slack 等默认采用 DM pairing 策略。
  • 应避免把智能体直接暴露在高权限、高风险、无审计的环境中。
  • 教学场景下要特别避免上传学生隐私数据与敏感校务材料。
# README 提到的思路 - unknown senders 先收到 pairing code - 需通过本地 approve 加入 allowlist - real messaging surfaces = untrusted input 教育现场可转化为： 先小范围测试，再逐步开放权限
安全边界先控风险，再谈规模化

OpenClaw 之所以值得带进教师工作坊，不在于它“火”，而在于它让我们第一次可以比较真实地把本地、持续、可行动的教育智能体带到老师面前。

本地部署 持久记忆 渠道接入 技能扩展 教育场景定制
